Idea that started the project

My friends and I used to play a lot of Counter-Strike 2, and during that time we purchased various in-game cases, which could later be sold for profit. I wanted to create a platform where we could track the value of our inventories, monitor price changes over time, and determine the best moment to sell. This project also served as a valuable opportunity for me to improve my development skills and learn new technologies.

More about the project

This web application was built using the Flask framework and serves as a tool for monitoring the prices of CS2 in-game cases available on the Steam market. Users can view historical price charts, track day-to-day, week-to-week, month-to-month and total value changes, and register accounts to manage their personal inventories. The platform automatically calculates the total value of owned cases and visualizes market fluctuations to assist in identifying optimal selling times.

The project utilizes a combination of Python (Flask) for backend logic and routing, SQLAlchemy for database interactions, and HTML5, CSS3, JavaScript, and Bootstrap for a responsive and user-friendly frontend. For data visualization, it uses Chart.js. Price data is collected via the Steam Marketplace API and stored in JSON files to allow flexible access and to practice working with structured data formats. The application includes a user authentication system with registration and login functionality, where passwords are protected through secure hashing. It also features interactive forms that allow users to track their owned cases, monitor value changes over time, and calculate their potential profit. Version control is managed using Git and GitHub. This project not only addresses a real-world need for players but also served as a comprehensive learning experience in full-stack web development.

Home

The homepage displays a dynamically generated list of all Counter-Strike 2 cases available on the Steam Market, each represented by a card containing the case name, image fetched from the Steam API, and the latest market price. Clicking on a case redirects the user to a detailed subpage showing a historical price chart and metadata. Data is regularly fetched and stored in JSON format for performance optimization and to demonstrate API handling and data processing.

User Chests

The user dashboard enables authenticated users to input and manage their owned Counter-Strike 2 cases along with quantities. Each entry is displayed in a dynamic, responsive table that includes the case’s latest market price, total value (with and without Steam transaction fees), and percentage change from the previous day. Users can update or remove individual case records. Below the table, the application calculates and displays the cumulative value of all listed cases, both pre- and post-tax.

Chest Page

The chest detail page presents an interactive price history chart powered by Chart.js, allowing users to analyze case price fluctuations over various time ranges, including the past week, month, year, and all-time. Users can dynamically select the desired time frame. Below the chart, a data table displays historical prices by date along with the corresponding percentage change compared to the previous recorded value.

Calculator

The calculator is designed for users who prefer not to create an account. It offers similar functionality to the user inventory page, allowing users to input the cases they own and their respective quantities. However, it does not persist any data — once the session ends or the page is refreshed, all information is cleared. This tool provides a quick way to estimate the total value of owned cases, both before and after Steam transaction fees.

Login/Register

The project includes a secure user authentication system featuring user registration and login functionality. During registration, user credentials are validated and passwords are securely hashed using industry-standard techniques before being stored in the database. Upon logging in, users gain access to personalized features such as managing their own case inventory. Session management is handled via Flask’s session mechanism to maintain user state throughout the visit.
